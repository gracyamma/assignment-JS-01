  ## Q20. Explain the client-server architecture

  ANS:Imagine you are using your phone and you open a website.
Your phone or browser acts as a client. The moment you type a website address and press enter, the client sends a request saying, “I want this webpage.”
That request travels through the internet and reaches a powerful computer called a server. The server is always waiting for such requests. When it receives your request, it processes it—this may involve fetching files, images, or data from a database and preparing the webpage.
Once everything is ready, the server sends the response back across the internet to your phone.
Your browser (client) receives the response and displays the webpage on your screen.
So, in simple narration:
You ask for something → the client sends the request → the server processes it → the server sends back the result → the client shows it to you.
This back-and-forth communication between client and server is called the client–server architecture.

### client

A client is the device or application that requests services or data.

eg:Web browser (Chrome, Firefox)

Mobile apps

Your computer or phone

### server

A server is a powerful system that provides services, processes requests, and sends responses.

eg:Web servers

Database servers

Application servers
